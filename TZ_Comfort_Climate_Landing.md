# ТЗ: Лендинг «Встановлення кондиціонерів та вікон» — Comfort Climate

**Версія:** 1.0  
**Дата:** 13.06.2026  
**Компанія:** Comfort Climate  
**Місто / регіон роботи:** `[ЗАПОВНИТИ місто та область]`  
**Розміщення:** окремий сайт або піддомен `[ЗАПОВНИТИ домен / піддомен]`  
**Мета лендингу:** отримання заявок на замір, консультацію, підбір рішення та прорахунок вартості для двох напрямів: **вікна** і **кондиціонери**.  
**Ключова особливість:** сайт має перемикатися між двома режимами — **«Вікна»** та **«Кондиціонери»**. Користувач бачить релевантний контент, CTA, візуал, блоки та форму залежно від обраного режиму.  
**Цільова дія:** заявка через форму + замовлення дзвінка + дзвінок. Перехід у месенджер — додатковий канал зв'язку, але **не основна конверсія**.

## 0. ЯК ЧИТАТИ ЦЕЙ ДОКУМЕНТ

- Це технічне завдання для розробки рекламного лендингу під платний трафік Google Ads / Meta Ads.
- Сайт має працювати як одна посадкова сторінка, але з двома окремими рекламними сценаріями:
  - **режим “Вікна”** — для трафіку на встановлення / заміну / ремонт вікон;
  - **режим “Кондиціонери”** — для трафіку на встановлення / сервіс / підбір кондиціонерів.
- Основний трафік — мобільний. Розробка **mobile-first**.
- На сайті має бути помітний перемикач режимів, але також мають працювати прямі рекламні посилання на конкретний напрям:
  - `/?mode=windows` або `/windows` — відкриває сайт одразу в режимі «Вікна»;
  - `/?mode=conditioners` або `/conditioners` — відкриває сайт одразу в режимі «Кондиціонери».
- Сайт повинен виглядати сучасно, технологічно, чисто й надійно. Без перевантаження текстом.
- Місця, позначені `[ЗАПОВНИТИ]`, не вигадувати. Це реальні дані, які має надати компанія.
- Тексти можна використовувати як фінальні, але перед запуском узгодити назву компанії, місто, ціни, гарантії, фото, сертифікати та умови робіт.

### 1.1 Режими сайту

Сайт має мати два активні стани:

**Режим 1 — Вікна**
- hero про встановлення / заміну вікон;
- візуал із вікном, світлом, теплом, шумоізоляцією;
- переваги, послуги, FAQ, ціни та форма під вікна;
- CTA: **«Замовити безкоштовний замір»** або **«Розрахувати вартість вікон»**.

**Режим 2 — Кондиціонери**
- hero про встановлення / підбір кондиціонера;
- візуал із потоком прохолодного повітря, комфортом у квартирі / офісі;

- CTA: **«Підібрати кондиціонер»** або **«Замовити монтаж»**.

> Важливо: перемикач має змінювати не лише заголовок, а весь ключовий контент сторінки: оффер, іконки, послуги, блок страхів, ціни, FAQ, форму, події аналітики.

## 2. КОЛІРНА СХЕМА ТА СТИЛЬ

Для цієї ніші краще використовувати чисту, технологічну й “свіжу” палітру. Вона має асоціюватися з прохолодою, чистим повітрям, енергоефективністю, теплом у домі та надійним монтажем.

### Основна палітра

- **Основний темний:** графітово-синій `#0F172A`  
  Використовується для заголовків, хедера, футера, темних секцій.

- **Основний світлий фон:** холодний білий `#F8FBFF`  
  Використовується як базовий фон сторінки.

- **Акцент для режиму “Вікна”:** чистий бірюзово-синій `#00A6A6`  
  Асоціації: скло, світло, енергоефективність, сучасні віконні системи.

- **Акцент для режиму “Кондиціонери”:** прохолодний блакитний `#1CA7EC`  
  Асоціації: холод, повітря, свіже приміщення, комфорт влітку.

- **CTA-кнопка:** теплий жовто-помаранчевий `#FFB703`  
  Має контрастувати з холодною палітрою й добре виділятися. Використовувати для головних кнопок.

- **Текст:** темно-сірий `#243042`  
  Не використовувати чистий чорний.

- **Другорядний текст:** `#64748B`

### Стиль

- Багато повітря, м'які тіні, сучасні картки.
- Іконки — лінійні або об'ємні, але без дешевого стокового вигляду.
- Візуальний стиль: **сучасний сервіс для дому**, не “будівельна бригада з OLX”.
- Уникати агресивних червоних кольорів, кислотних градієнтів, важких фонів і занадто темного дизайну.
- Для режимів можна використовувати м'яку зміну акценту:
  - windows: бірюзові лінії, відблиски скла, світло;
  - conditioners: блакитні хвилі повітря, легкий frost/glassmorphism.

## 3. ГЛОБАЛЬНІ ЕЛЕМЕНТИ

### 3.1 Sticky-хедер

Хедер закріплений зверху.

Елементи:
- логотип `[ЗАПОВНИТИ]`;
- перемикач режимів: **Вікна / Кондиціонери**;
- телефон, клікабельний на мобільному: `[ЗАПОВНИТИ телефон]`;
- кнопка CTA:
  - у режимі “Вікна”: **Замовити замір**;
  - у режимі “Кондиціонери”: **Замовити монтаж**.

На мобільному:
- телефон має бути видимим одразу;
- перемикач режимів можна зробити як компактний segmented control під хедером або всередині першого екрана;
- CTA не має перекривати контент.

### 3.2 Перемикач режимів

Формат: сегментований перемикач:

`[ Вікна ] [ Кондиціонери ]`

Поведінка:
- активний режим підсвічується відповідним акцентним кольором;
- при перемиканні змінюється контент без перезавантаження сторінки або з плавним переходом;
- URL має оновлюватися, щоб можна було дати пряме посилання на потрібний режим;
- вибір режиму зберігається в `localStorage`;
- якщо користувач прийшов з рекламного посилання `?mode=windows`, пріоритет має URL, а не попередній вибір.

### 3.3 Плаваюча мобільна CTA-кнопка

Після першого скролу з'являється нижня закріплена кнопка:

- режим “Вікна”: **Розрахувати вікна**;
- режим “Кондиціонери”: **Підібрати кондиціонер**.

Поруч може бути іконка дзвінка.

### 3.4 Футер

- Назва компанії;
- адреса / регіон роботи;
- телефон;
- email;
- графік роботи;
- месенджери: Telegram, Viber, WhatsApp;
- карта або список міст/районів обслуговування;
- реквізити ФОП / ТОВ `[ЗАПОВНИТИ за потреби]`;
- посилання на політику конфіденційності.

## 4. 3D-АНІМАЦІЯ ТА ВІЗУАЛЬНА ІДЕЯ

### 4.1 Загальна ідея

На першому екрані має бути легка 3D-анімація, яка змінюється залежно від режиму.

**У режимі “Вікна”:**
- 3D-модель вікна або склопакета;
- легке відкривання стулки;
- сонячне світло / тепловий контур;
- візуальний натяк на шумоізоляцію та тепло.

**У режимі “Кондиціонери”:**
- 3D-модель внутрішнього блоку кондиціонера;
- м'який потік повітря;
- легка анімація прохолодних хвиль;
- можна показати “кімнату до / після”: душно → комфортно.

### 4.2 Технічна реалізація 3D

Допустимі варіанти:

1. **Spline embed** — швидко і візуально красиво, але потрібно перевірити вагу.
2. **Three.js + glTF модель** — кращий контроль і оптимізація.
3. **Lottie / pseudo-3D** — найшвидший варіант, якщо бюджет обмежений.
4. **Відео WebM з alpha / fallback** — якщо 3D важко оптимізувати.

Вимоги:
- 3D не має блокувати LCP;
- на слабких телефонах має підставлятися статичне WebP-зображення;
- анімація має бути декоративною, не заважати заявці;
- максимальна вага 3D-моделі після оптимізації — бажано до 1–1.5 МБ;
- якщо 3D не завантажилось за 2 секунди — показати fallback.

**Для “Вікна”:**

## БЛОК 3 — “ЩО ВАМ ПОТРІБНО?” / МІНІ-КВІЗ

Це швидкий блок для сегментації користувача. Він має допомогти людині швидше перейти до заявки.

### У режимі “Вікна”

**Заголовок:**
> Що потрібно зробити з вікнами?

Картки-вибір:
- замінити старі вікна в квартирі;
- встановити вікна в будинку;
- засклити балкон або лоджію;
- підібрати енергоефективні вікна;
- замінити лише склопакет / фурнітуру;
- потрібен замір і прорахунок.

### У режимі “Кондиціонери”

**Заголовок:**
> Для чого потрібен кондиціонер?

Картки-вибір:
- квартира;
- приватний будинок;
- офіс;
- магазин / салон;
- серверна / технічне приміщення;
- потрібно обслуговування або демонтаж.

Після вибору можна:
- плавно прокрутити до форми;
- підставити обраний запит у приховане поле форми;
- передати вибір у Telegram-повідомлення та GA4 event.

## БЛОК 4 — БОЛІ ТА СТРАХИ

Цей блок має закривати сумніви користувача. Формат: картки “страх → відповідь”.

### Режим “Вікна”

**Заголовок:**
> Найчастіше люди бояться не вікон, а поганого монтажу

**Картки:**

**«Боюся, що після заміни буде дути»**  
> Тепло залежить не лише від профілю, а й від монтажу. Ми звертаємо увагу на замір, монтажний шов, герметизацію і правильне встановлення під конкретний отвір.

**«Не хочу переплачувати за непотрібний профіль»**  
> Ми не продаємо найдорожче “бо так краще”. Пояснюємо різницю між варіантами й допомагаємо вибрати рішення під вашу квартиру, будинок і бюджет.

**«Боюся бруду та ремонту після встановлення»**  
> Монтаж вікон — це робота з пилом і демонтажем, але її можна зробити акуратно. Захищаємо зону робіт, попереджаємо, що потрібно підготувати, і прибираємо основне після монтажу.

**«Не розумію, яка ціна буде в результаті»**  
> Остаточна вартість залежить від розміру, профілю, склопакета, фурнітури й монтажу. Саме тому потрібен замір — після нього ви отримуєте зрозумілий прорахунок.

### Режим “Кондиціонери”

**Заголовок:**
> Хороший кондиціонер — це не тільки бренд, а правильний підбір і монтаж

**Картки:**

**«Боюся купити слабкий або занадто потужний кондиціонер»**  
> Ми підбираємо модель за площею, висотою стелі, сонячною стороною, кількістю людей і техніки в кімнаті. Не “на око”, а під реальні умови.

**«Не хочу, щоб зіпсували ремонт»**  
> Перед монтажем узгоджуємо місце внутрішнього й зовнішнього блоків, трасу та відведення конденсату. Працюємо акуратно й пояснюємо, де будуть отвори та комунікації.

**«Боюся, що кондиціонер буде шуміти або текти»**  
> Такі проблеми часто виникають через неправильний монтаж, відведення конденсату або помилки запуску. Ми перевіряємо систему після встановлення й пояснюємо правила користування.

**«Не знаю, який бренд обрати»**  
> Ми допоможемо порівняти варіанти за бюджетом, гарантією, шумом, енергоефективністю та сервісом. Остаточне рішення залишається за вами.

## 8. РЕКЛАМНІ СЦЕНАРІЇ ТА ПОСАДКОВІ СЕКЦІЇ

Потрібно зробити так, щоб рекламу можна було заливати окремо на кожен напрям.

### Варіант 1 — через URL-параметр

- `site.com/?mode=windows&utm_source=google&utm_campaign=windows_install`
- `site.com/?mode=conditioners&utm_source=google&utm_campaign=ac_install`

### Варіант 2 — через окремі URL

- `site.com/windows`
- `site.com/conditioners`

Обидва варіанти можуть вести на один і той самий код, але відкривати різний режим.

### Для реклами на вікна

Перший екран має одразу показувати:
- заголовок про вікна;
- 3D-вікно;
- CTA “Замовити замір”;
- блоки та FAQ про вікна;
- форма з прихованим полем `lead_type=windows`.

### Для реклами на кондиціонери

Перший екран має одразу показувати:
- заголовок про кондиціонери;
- 3D-кондиціонер;
- CTA “Підібрати кондиціонер”;
- блоки та FAQ про кондиціонери;
- форма з прихованим полем `lead_type=conditioners`.

> Розробнику: не можна вести користувача з реклами на “загальний” hero, де він сам має ще раз обирати напрям. Якщо реклама про кондиціонери — сайт відкривається в режимі кондиціонерів. Якщо реклама про вікна — у режимі вікон.

## 15. КОРОТКА ЛОГІКА ДЛЯ РОЗРОБНИКА

1. При відкритті сторінки перевірити URL:
   - якщо `?mode=windows` або `/windows` → активувати режим “Вікна”;
   - якщо `?mode=conditioners` або `/conditioners` → активувати режим “Кондиціонери”;
   - якщо режиму немає → показати дефолтний режим `[УЗГОДИТИ: бажано той, на який буде більше реклами]`.

2. При перемиканні режиму:
   - змінити hero;
   - змінити accent color;
   - змінити 3D-візуал;
   - змінити послуги;
   - змінити FAQ;
   - змінити форму;
   - оновити URL;
   - зберегти режим у `localStorage`;
   - відправити event `mode_switch`.

3. При відправці форми:
   - валідувати телефон;
   - зберегти заявку в резерв;
   - відправити в Telegram;
   - відправити `form_submit` в GTM;
   - показати користувачу повідомлення:  
     > Дякуємо, ми отримали заявку й скоро з вами зв'яжемося.
